# 配置驱动设计说明

## 问题说明

之前 `data_processor.py` 中硬编码了数据过滤规则：
```python
# 硬编码的过滤规则 ❌
teams_to_remove = ['安全內控團隊', '前端網路團隊', '系統平臺團隊']
df = df[df['工单状态'] != '已撤銷']
```

这种做法的问题：
- ❌ 规则写死在代码中，修改需要重新编程
- ❌ 不符合配置驱动的设计理念
- ❌ 与 `transform_rules.md` 的描述不一致
- ❌ 维护困难，扩展性差

## 解决方案

### 1. 配置文件 (`config.ini`)

在 `config.ini` 中添加了 `[filter]` 部分：

```ini
[filter]
# 数据过滤规则
# 移除工单状态为以下值的行（逗号分隔）
removed_status = 已撤銷

# 移除提单所属团队为以下值的行（逗号分隔）
removed_teams = 安全內控團隊,前端網路團隊,系統平臺團隊

# 是否移除'序号'列
remove_serial_column = true
```

### 2. 代码改动

**data_processor.py** 改动：

1. **`load_config()` 函数** - 增加过滤规则读取：
```python
# 读取过滤规则配置
removed_status = config.get('filter', 'removed_status', fallback='已撤銷')
removed_teams_str = config.get('filter', 'removed_teams', fallback='安全內控團隊,前端網路團隊,系統平臺團隊')
removed_teams = [t.strip() for t in removed_teams_str.split(',') if t.strip()]
remove_serial_column = config.getboolean('filter', 'remove_serial_column', fallback=True)
```

2. **`process_source_data()` 函数** - 使用配置中的规则：
```python
# 从配置获取过滤规则
if config is None:
    config = load_config()

# 移除指定列（如'序号'）
if config.get('remove_serial_column', True):
    if '序号' in df.columns:
        df = df.drop(columns=['序号'])
        print(f"  ✓ Removed '序号' column (per config)")

# 过滤工单状态（从配置读取）
removed_status = config.get('removed_status', '已撤銷')
if removed_status and '工单状态' in df.columns:
    df = df[df['工单状态'] != removed_status]
    print(f"  ✓ Removed rows with status '{removed_status}' (per config)")

# 过滤提单所属团队（从配置读取）
removed_teams = config.get('removed_teams', [])
if removed_teams and '提單人所屬團隊' in df.columns:
    df = df[~df['提單人所屬團隊'].isin(removed_teams)]
    print(f"  ✓ Removed rows with teams in {removed_teams} (per config)")
```

### 3. 文档更新

**transform_rules.md** 更新：

```markdown
#### filtered_source_data 数组
- **数据过滤规则**（在 `config.ini` 的 `[filter]` 部分配置）:
  - `removed_status`: 移除指定工单状态的行（默认：已撤銷）
  - `removed_teams`: 移除指定提单所属团队的行
  - `remove_serial_column`: 是否移除'序号'列（默认：true）

**注**: 所有过滤规则均可通过修改 `config.ini` 文件进行调整，无需修改代码
```

## 优势

### ✅ 配置驱动
- 规则在配置文件中，易于修改
- 无需重新编程，只需编辑配置
- 符合软件工程最佳实践

### ✅ 灵活性
- 可以随时添加/删除过滤规则
- 支持逗号分隔的多值配置
- 可选的布尔开关

### ✅ 可维护性
- 配置和代码分离
- 规则集中管理
- 易于理解和扩展

### ✅ 一致性
- 代码、配置、文档保持一致
- 所有规则来源统一（config.ini）
- transform_rules.md 准确描述规则来源

## 如何修改过滤规则

### 方法1: 编辑 config.ini

直接编辑 `config.ini` 文件：

```ini
[filter]
# 示例：修改过滤规则
removed_status = 已撤銷,草稿  # 支持多个状态
removed_teams = 安全內控團隊,前端網路團隊,系統平臺團隊,测试团队
remove_serial_column = true
```

### 方法2: 禁用某些过滤

```ini
[filter]
# 禁用团队过滤
removed_status = 已撤銷
removed_teams =  # 留空表示不过滤团队
remove_serial_column = true
```

### 方法3: 运行时查看效果

运行程序后查看日志输出：
```
✓ Removed '序号' column (per config)
✓ Removed 1 rows with '工单状态' = '已撤銷' (per config)
✓ Removed 5 rows with '提單人所屬團隊' in ['安全內控團隊', '前端網路團隊', '系統平臺團隊'] (per config)
```

所有带有 `(per config)` 标记的规则都是从配置文件读取的。

## 总结

通过这次改动：
- ✅ 消除了硬编码
- ✅ 实现了配置驱动设计
- ✅ 提高了可维护性和灵活性
- ✅ 代码、配置、文档保持一致

现在 `data_processor.py` 的作用更加清晰：
1. 读取 Excel 数据
2. **应用配置文件中的过滤规则**
3. 转换为 AI 友好的文本格式

过滤规则不再硬编码，而是完全由 `config.ini` 控制！
