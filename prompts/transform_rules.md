# 排班表转换规则

## 概述
此规则描述如何将"变更明细数据报表"转换为排班表

**核心原则**：
1. 工单必须竖着排列，每个工单占一行
2. 不能有任何空白单元格
3. 保留所有源数据列（在filtered_source_data中）
4. 确保所有日期都被处理

## 输入输出

### 输入数据
- 数据源: Excel 文本格式（由 Python 转换提供）
- 数据范围: 所有有效的变更工单
- 字段包括: 工單號, 變更系統名稱匯總, 變更名稱, 提單人, 計劃開始時間, 計劃結束時間

### 输出格式
- 输出为 JSON 格式
- 包含排班表结构和过滤后的源数据

## 数据字段映射

| 源字段 | 目标用途 | 说明 |
|--------|----------|------|
| 工單號 | 工单号 | 提取最后4位 |
| 變更系統名稱匯總 | 英文缩写 | 提取系统代码（如GMC） |
| 變更名稱 | 英文缩写 | 备用提取来源 |
| 提單人 | 中文姓名 | 提取斜杠前的姓名 |
| 計劃開始時間 | 任务时间 | 分组依据，用于判断特殊情况 |
| 計劃結束時間 | 任务时间 | 用于计算时间范围和间隔 |

## 转换规则

### 1. 按日期分组
- 字段: `計劃開始時間`
- 格式: 中文日期格式（如"5月04日"、"5月10日"）
- 排序: 按日期升序
- **重要**: 必须处理所有日期，不能遗漏任何一天

### 2. 排班表布局规则

#### Excel结构说明
排班表从第2行开始（第1行是标题）：
- **第2行（row_index=1）**: 所有日期，水平排列
- **第3行（row_index=2）**: 时间范围或时间点，水平排列
- **第4行及以后（row_index≥3）**: 工单信息，**竖着排列**

#### 关键规则：工单竖着排列

**错误示例**（横向排列）：
```
5月07日
18:00 至 20:00
(中台) 2241 CCM 晏鵬禹  (中台) 2237 TBU 王文斌  (中台) 2230 CRM 王文斌  ...
```

**正确示例**（竖向排列）：
```
5月07日
18:00 至 20:00
(中台) 2241 CCM 晏鵬禹
(中台) 2237 TBU 王文斌
(中台) 2230 CRM 王文斌
(中台) 2231 ATM 廖智堅
(中台) 2228 MFT 沈彪
(中台) 2214 SDB 何子楓
```

### 3. 时间处理规则

#### 普通情况：工作日且无时间间隔
**触发条件**:
- 周一、周二、周三、周四、周五
- 且 任务间无时间间隔（<1小时）

**输出格式**:
- **日期行**: 日期占一列
- **时间行**: 时间范围（最早开始 至 最晚结束）
  - 格式: "HH:MM 至 HH:MM"
  - 例如: "18:00 至 20:00"
- **工单行**: 每个工单占一行，**竖着排列**
  - 格式: "(中台) {工单号后4位} {英文缩写} {中文姓名}"
  - 对齐: 居中
  - **重要**: 工单数 = 该日期的行数（从第4行开始）

**示例**:
```
5月07日          5月09日
18:00 至 20:00   12:00 至 12:30
(中台) 2241 CCM 晏鵬禹    (中台) 2167 MGS 陳智龍
(中台) 2237 TBU 王文斌    NaN
(中台) 2230 CRM 王文斌    NaN
(中台) 2231 ATM 廖智堅    NaN
(中台) 2228 MFT 沈彪     NaN
(中台) 2214 SDB 何子楓    NaN
```

**注意事项**:
- 每个工单占一行，不是横向合并
- 不同日期的工单数可能不同，这是正常的
- 使用NaN填充空白区域（Python会处理为空单元格）

#### 特殊情况：周末任务 或 有时间间隔≥1小时的任务
**触发条件**:
- 周末（周六或周日）
- 或者任务间有≥1小时的时间间隔

**输出格式**:
- **日期行**: 日期占一列
- **时间行**: 显示所有时间点（开始时间和结束时间）
  - 格式: "HH:MM"（24小时制）
  - 排序: 按时间升序
  - 例如: "01:00", "02:00", "03:00", "06:00", "10:00", "11:00"
- **工单行**: 每个工单占一行，**竖着排列**
  - 格式: "(中台) {工单号后4位} {英文缩写} {中文姓名}"
  - 对齐: 居中

**示例**:
```
5月10日
01:00   02:00   03:00   06:00   10:00   11:00
(中台) 2197 AAS 張雋宇    NaN     NaN     NaN     NaN     NaN
(中台) 2224 GSW 廖智堅    NaN     NaN     NaN     NaN     NaN
(中台) 2234 CCS 楊行健    NaN     NaN     NaN     NaN     NaN
```

**特殊处理逻辑**:
1. 收集所有任务的所有时间点（开始时间和结束时间）
2. 对时间点进行排序
3. 时间行：每个时间点占一列
4. 工单行：每个工单占一行，根据时间跨度和人员信息填充

### 4. 辅助函数说明

#### extract_english_abbr(system_name, change_name)
从系统名称或变更名称中提取英文缩写
- 优先从"變更名稱"提取（匹配2个或更多连续的大写字母）
- 若未找到，从"系統名稱"提取
- 若都未找到，返回"XXX"

示例:
- "GMC-GMC列印系統" -> "GMC"
- "CCP-人民币跨境支付系统" -> "CCP"
- "MFT-數據傳輸平台" -> "MFT"
- "RMS-招聘管理系統" -> "RMS"
- "MEG-系統" -> "MEG"
- "TBU-大數據平台" -> "TBU"
- "CRM-零售客戶關係管理系統" -> "CRM"
- "ATM-GRG-GRG ATM終端應用程序" -> "ATM"
- "AAS" -> "AAS"
- "CCS-清算行系統" -> "CCS"
- "GSW" -> "GSW"
- "SDB" -> "SDB"
- "MGS" -> "MGS"

#### extract_chinese_name(submitter)
从提單人字段中提取中文姓名
- 格式: "姓名/ID" -> "姓名"
- 若无斜杠，返回原字符串

#### format_work_order_item(row)
格式化工单项目
- 格式: "(中台) {工单号后4位} {英文缩写} {中文姓名}"
- 示例: "(中台) 2249 RMS 黃懌靜"

#### format_time_window(start_time, end_time)
格式化时间窗口
- 格式: "HH:MM 至 HH:MM"
- 示例: "18:00 至 19:00", "09:00 至 10:00"

#### format_chinese_date(date_obj)
格式化中文日期
- 格式: "{月份}月{日期}日"
- 日期补零: "5月04日", "5月10日"
- **重要**: 使用5月而不是05月

#### is_weekend(date_val)
判断是否为周末
- 周六: weekday() = 5
- 周日: weekday() = 6
- 返回: True 或 False

#### has_time_gaps(sorted_group, min_gap_hours=1)
检查任务间是否存在时间间隔
- 参数:
  - sorted_group: 按計劃開始時間排序的任务组
  - min_gap_hours: 最小间隔小时数（默认1）
- 返回: True（有间隔）或 False（无间隔）

#### is_thursday(date_val)
判断是否为周四
- 周四: weekday() = 3
- 返回: True 或 False

#### distribute_workorders_evenly_with_grouping(items, target_count)
将工单均匀分配到指定数量的单元格中
- 参数:
  - items: 工单信息列表，每个元素包含 {'text': 工单文本, 'person': 人员姓名}
  - target_count: 目标单元格数量（来自配置的 thursday_person_count）
- 逻辑:
  1. 如果 target_count >= 人员数 → 直接均匀分配（不合并同一人的工单）
  2. 如果 target_count < 人员数 → 同一人的工单合并，其他人均匀分配

### 字段说明

#### schedule 数组
- **date**: 日期字符串（中文格式）
- **date_merge_range**: 日期单元格位置（Excel 格式，如 "B2:B2"）
- **type**: 日期类型
  - "normal": 普通情况（工作日且无时间间隔）
  - "special": 特殊情况（周末或有时间间隔）
- **time_range**: 时间范围字符串（仅 normal 类型）
  - 格式: "HH:MM 至 HH:MM"
- **time_points**: 时间点数组（仅 special 类型）
  - 按升序排列的所有时间点
  - 格式: "HH:MM"
- **work_orders**: 工单数组
  - **person**: 人员姓名
  - **merge_range**: 单元格位置（Excel 格式，如 "B4"）
  - **content**: 单元格内容

#### filtered_source_data 数组
- **重要**: 必须保留所有列！
- 过滤后的源数据（用于第二个 sheet）
- 包含所有字段（根据配置去除指定列，如"序号"）
- **数据过滤规则**（在 `config.ini` 的 `[filter]` 部分配置）:
  - `removed_status`: 移除指定工单状态的行（默认：已撤銷）
  - `removed_teams`: 移除指定提单所属团队的行（默认：安全內控團隊,前端網路團隊,系統平臺團隊）
  - `remove_serial_column`: 是否移除'序号'列（默认：true）

**注**: 所有过滤规则均可通过修改 `config.ini` 文件进行调整，无需修改代码

## 数据处理流程

1. 加载 `config.ini` 配置文件（包含过滤规则）
2. 读取 Excel 数据文件
3. **应用数据过滤**（根据 `config.ini` 的 `[filter]` 配置）:
   - 移除指定列（如'序号'）
   - 过滤指定工单状态的行（如'已撤銷'）
   - 过滤指定提单所属团队的行（如'安全內控團隊'等）
4. 将过滤后的数据转换为AI友好的文本格式
5. 接收文本格式的 Excel 数据
6. 提取列头和数据行
7. 清理空行
8. 过滤无效日期（NaT）
9. 解析日期时间字段
10. 按日期分组
11. **重要**: 确保所有日期都被处理，不能遗漏
12. 对每组判断是否为特殊情况（周末或时间间隔）
13. 应用相应的转换规则：
    - 特殊情况：显示所有时间点，每个工单竖着排列
    - 普通情况：显示时间范围，每个工单竖着排列
14. 生成 JSON 结构输出
15. 生成源数据sheet（保留所有列）

## 重要注意事项

1. **工单必须竖着排列**：每个工单占一行，不是横向合并
2. **不能有空白单元格**：所有数据必须填充完整
3. **保留所有源数据列**：在filtered_source_data中
4. **处理所有日期**：不能遗漏任何一天
5. **日期格式**：使用"5月04日"而不是"05月04日"
6. **时间格式**：使用"18:00"而不是"18:00:00"
7. **工单号提取**：提取最后4位，如"2249"
8. **系统缩写**：从系统名称中提取大写字母部分

## 错误处理

- 无效日期: 过滤掉
- 空工单号: 忽略该行
- 缺少字段: 使用默认值或空字符串
- 文件不存在: 显示错误并退出
