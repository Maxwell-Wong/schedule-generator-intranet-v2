# Prompt模板说明

## 文件位置
`prompts/ai_prompt_template.txt`

## 用途
这个文件包含了发送给AI的完整提示词模板。你可以通过修改这个文件来调整AI的行为，而不需要修改Python代码。

## 占位符说明

模板中包含以下占位符，程序会自动替换：

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{RULES_TEXT}` | 转换规则内容 | 从 `prompts/transform_rules.md` 读取 |
| `{DATA_TEXT}` | Excel数据文本 | 从Excel文件转换而来 |
| `{THURSDAY_COUNT}` | 周四人数参数 | 从config.ini读取（默认6） |

## 修改Prompt的建议

### 1. 修改输出要求
如果你想改变AI的输出格式，可以修改"输出要求"部分：

```
# 输出要求
请返回纯JSON格式（不要使用markdown代码块），直接输出JSON内容。

重要提示：
1. 周末任务（周六、周日）或时间间隔≥1小时的任务，type设为"special"
2. 特殊情况需要显示所有时间点（time_points数组）
3. 普通情况只显示时间范围（time_range字符串）
4. 周四的工单需要根据thursday_person_count参数均匀分配
5. 非周四每格最多3个工单
6. merge_range使用Excel格式（如"D2:F2", "D4:F4"）
7. 日期格式使用中文（如"1月05日"）
8. 时间格式使用24小时制（如"0:30", "2:00"）
9. 确保JSON格式正确，所有必需字段都存在
```

### 2. 添加额外指令
如果你想让AI更严格地遵循某些规则，可以在"重要提示"部分添加：

```
重要提示：
...
10. 不要遗漏任何工单
11. 确保每个工单都有正确的人员信息
12. 如果遇到无法解析的数据，请跳过该行并记录日志
```

### 3. 修改角色定义
如果你想改变AI的角色定位：

```
# 角色定义
你是一个专业的数据处理助手，负责将变更明细数据转换为排班表结构。
你需要像有10年经验的排班管理员一样仔细处理数据。
```

### 4. 添加示例
你可以在prompt中添加示例，帮助AI更好地理解：

```
# 示例输出
{
  "schedule": [
    {
      "date": "1月05日",
      "date_merge_range": "D2:E2",
      "type": "normal",
      "time_range": "0:30 至 3:30",
      "work_orders": [...]
    }
  ]
}
```

## 注意事项

1. **保留占位符**：不要删除 `{RULES_TEXT}`, `{DATA_TEXT}`, `{THURSDAY_COUNT}` 这些占位符
2. **保持结构**：建议保持现有的prompt结构（角色定义、任务说明、转换规则、输入数据、输出要求）
3. **测试修改**：修改后运行 `test_flow.py` 或 `main.py` 来验证效果
4. **备份原文件**：修改前建议备份原始模板

## 相关文件

- `prompts/transform_rules.md` - 转换规则（业务逻辑）
- `schemas/output_schema.json` - JSON Schema定义
- `ai_client.py` - AI客户端代码（读取此模板）

## 调试技巧

如果AI的输出不符合预期：

1. **查看完整Prompt**：在 `ai_client.py` 中添加调试代码打印完整prompt
2. **简化Prompt**：先简化prompt测试AI理解能力
3. **逐步添加**：逐步添加指令，找到导致问题的部分
4. **使用示例**：在prompt中添加具体的输入输出示例

## 版本控制

建议将修改后的prompt模板纳入版本控制，以便：
- 追踪修改历史
- 对比不同版本的效果
- 回退到之前的版本
